What is imapex 101?

Developing applications using modern technologies and practices like DevOps requires more than simply understanding a programming language like Python or Javascript. Experience and skill with source control, container technologies, collaboration services, and build bots is also needed.

imapex 101 is an intensive hands-on training program designed to jump start new developers with a foundation in DevOps practices. The training material is designed to introduce a concept, provide basic hands-on examples, and challenges to reinforce the material.

The learning modules and labs used within imapex 101 are a subset of training material developed by the imapex community. All modules and labs are available at github.com/imapex-training.

imapex 101 Modules

General Development Concepts

Approximately 90 minutes

Developing software involves much more than simply programming languages and APIs. In this module we will explore four topics that are fundamental to any successful development project.

General Development Skills

Approximately 120 minutes

This hands on module is designed to provide you with some experience with several important utility tools and technologies which are very important and time saving. We will cover Basic Linux Tools, Markdown, and several Python topics.

Advanced Git and GitHub

Approximately 180 minutes

Git and GitHub remain the most popular source control platforms for developers today. We will review the basics of git repository creation and interaction, before moving into topics including branches, pull requests, issues, and forking in this hands-on module.

Advanced Docker and Docker Hub

Approximately 120 minutes

Docker, continues to be a major aspect of modern development practices and deployment strategies. Topics discussed in this hands-on module include Dockerfile creation, registry options, the meaning and use of tags, and auto-build repositories.

Implementing CICD

Approximately 120 minutes

In the session we will dive into CICD and come out with a detailed understanding of what happens in a "Code Pipeline". The module begins with foundational knowledge about CICD, and end with a guided lab where a full pipeline is constructed and tested.

Deploying Micro-Service Applications

Approximately 90 minutes

Application developers need a modern infrastructure stack (or cloud) to deploy onto. In this module students are shown how to consume several leading Open Source tools including Marathon/Mesos, Consul and Traefik for deploying their applications onto a Mantl.io stack.

From Idea to Application

Approximately 60 minutes

Good demo applications should be easy to deploy, use and contribute code to. This means focus up front on questions like "How will others deploy my application?", "How can I quickly and easily test application changes?", and "What is my MVP and how can I build it fast!" are critical. In this module we'll suggest an approach to the building applications.

Suggested imapex 101 Agenda

Finding time to set aside for training can be difficult, so the imapex 101 program has been designed to be compact and intensive. Each module quickly covers the relevant concepts and spends the marjority of time with guided hands on lessons and exercises. Though each organizer can adjust as needed, the suggested agenda targets a 3 day training block.

* Times listed are only guidelines. Each module could go shorter or longer depending on discussion and challenges

Day 1

  • General Development Concepts 90m
  • General Development Skills 120m
  • Advanced Git and GitHub 180m

Day 2

  • Advanced Docker and Docker Hub 120m
  • Implementing CICD 120m
  • Deploying Micro-Service Applications 90m
  • Open Mic - Share Your Work 90m

Ending Day 2 by bringing in food and drinks during the "Open Mic" session is a great way to decompress after 2 solid days of focused work.

Day 3

  • Demo Brainstorming Session 60m
  • From Idea to Application 60m
  • Begin Demo Development 240m
